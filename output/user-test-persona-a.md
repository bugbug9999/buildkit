## 페르소나 프로필

- **이름:** 페르소나 A (30대 직장인 여성)
- **복약 현황:**
  - 위고비 0.5mg 피하주사 — 주 1회 금요일
  - 드시모네 1정 — 매일 아침
  - 항우울제 — 매일 아침 8시 / 저녁 22시
- **행동 특성:** 스마트폰 앱 자주 사용하나 복잡한 UI 싫어함 / 바로 이탈 / 병원 다음 날 약 빼먹는 경우 잦음

> **주의:** 기획서의 레퍼런스 유저(위고비 토요일)와 이 페르소나(위고비 금요일 + 항우울제 저녁 22시)는 다소 다름. 분석에서 이 차이를 명시적으로 다룸.

## 시나리오 1 — 첫 방문: 앱 투약 탭 처음 열기

### 실제 UI 구조 (코드 기반)

`medications.length === 0`이면 Empty State를 렌더링:
- 헤더: "투약 관리" + 우측 "약 관리" 링크
- 중앙 카드: 💊 아이콘 + "복용 중인 약을 등록해 주세요" + 설명 텍스트 + **[약 등록하기]** 버튼

### 페르소나 A의 행동

앱을 처음 열면 중앙에 큰 카드가 보인다. "복용 중인 약을 등록해 주세요"라는 명확한 문구와 초록색 CTA 버튼이 눈에 들어온다. 바로 [약 등록하기]를 누른다.

### 잘 된 부분

- 빈 화면이 아니라 명확한 행동 유도 — CTA 버튼이 즉시 보임
- "오늘 복약 체크, 달력 기록, 주간 요약을 한 번에 관리" 설명이 가치를 전달
- 기획서 Empty State와 실제 코드가 일치함

### 혼란 포인트

- 우측 상단 "약 관리" 텍스트 링크가 CTA 버튼과 같은 목적지인지 모름 → 중복처럼 보임
- "투약 관리"라는 탭 제목이 약 먹는 것인지 주사 관련인지 모호함 (기존 탭명이 "주사" 였을 가능성, injection.tsx 파일명 기준)
- 약 등록 안 하고 나갔다가 다시 왔을 때 같은 Empty State → 온보딩이 1회성인지 반복인지 불명확

## 시나리오 2 — 약 등록: 위고비 + 드시모네 + 항우울제 3개 등록

### 실제 UI 구조 (AddScreen 코드 기반)

약 등록 폼 필드 순서:
1. 약 이름 (TextInput)
2. 복용 형태 — 경구 / 주사 / 기타 (3버튼 Segment)
3. 용량 (TextInput, placeholder: "예: 0.5mg, 1정, 2캡슐")
4. 복용 주기 — 매일 / 요일 지정 (2버튼 Segment)
   - "요일 지정" 선택 시 월~일 요일 칩 노출
5. 복용 시간대 — 등록된 시간대 카드 + [+ 시간대 추가] 버튼
   - 추가 버튼 누르면 바텀시트 Modal 오픈
   - 바텀시트: 빠른 선택(아침/점심/저녁/취침 전) + 직접 입력(stepper)
6. 복용 조건 — 공복 / 식전 / 식후 / 무관 (4버튼)
7. 메모 (multiline TextInput)
8. 우측 상단 [저장] 버튼 (이름+시간대 없으면 비활성)

### 2-1. 위고비 등록

**동작:** 약 이름에 "위고비" 입력 → 형태에서 "💉 주사" 선택 → 용량 "0.5" 입력 → 복용 주기 "요일 지정" 선택 → "금" 칩 선택 → [+ 시간대 추가] 터치 → 바텀시트에서 "아침" 선택

**혼란 포인트:**
- "요일 지정" 선택 시 요일 칩이 나타나는 UX는 자연스럽지만, **요일 칩이 선택됐는지 안 됐는지 시각적 피드백이 작을 수 있음** (active 스타일 변화가 텍스트 색상 변화이므로)
- 바텀시트의 "빠른 선택"과 "직접 입력"이 같은 화면에 공존 — 이 페르소나는 빠른 선택의 "아침 08:00"을 누르면 바텀시트가 닫히며 추가됨. 직관적임
- **핵심 문제: 복용 주기가 "매일" / "요일 지정" 두 가지만 있음**. 위고비는 주 1회인데 "요일 지정"으로 선택해야 함. 레이블이 "주 N회"나 "주 1회"가 아니라 "요일 지정"이라 메탈 모델 불일치 발생
- 저장 후 다시 약 등록 화면으로 돌아가는 것이 아니라 메인으로 이동 (`setScreen("main")`) → 두 번째 약 등록을 위해 다시 [약 등록하기] 경로를 찾아야 함

**잘 된 부분:** 형태 선택(경구/주사/기타)이 버튼으로 명확하게 구분됨

### 2-2. 드시모네 등록

**동작:** 약 이름 "드시모네" → 경구 → "1정" → 매일 → [+ 시간대 추가] → 아침 → 식후 → 저장

**혼란 포인트:**
- 저장 후 다시 메인으로 이동 → 연속 등록 의도인데 네비게이션이 끊김. "약 관리" → "+ 새 약 등록하기" 버튼을 다시 찾아야 함 (ManageScreen → AddScreen 경로)
- **기획서에는 "다른 약도 등록할까요?" 프롬프트가 있는데 코드에 없음**

### 2-3. 항우울제 등록 (핵심 엣지 케이스)

**동작:** 약 이름 "항우울제" → 경구 → 매일 → [+ 시간대 추가] 터치

**바텀시트에서의 경험:**
- "빠른 선택"에서 "아침 08:00" 선택 → 추가 완료
- 다시 [+ 시간대 추가] 터치 → 바텀시트 재오픈
- 이제 "아침"은 이미 추가되어 있으므로 비활성(disabled) 처리됨
- "저녁 19:00" 선택 → 추가

**핵심 혼란 포인트:**
- **항우울제 복용 시간이 저녁 22시인데, 바텀시트 빠른 선택의 "취침 전" 프리셋은 22:00이 아니라 코드상 명시되지 않음** (`TIME_SLOT_PRESETS` 확인 필요, 스펙에는 취침 전이 있음)
  - 실제 코드: `TIME_SLOT_PRESETS = { morning: "08:00", lunch: "12:00", evening: "19:00", bedtime: "22:00" }` → 취침 전 22:00 프리셋 존재. 이건 맞음
  - 그러나 "저녁"(19:00)과 "취침 전"(22:00)이 둘 다 있어서 항우울제를 "저녁 22시"에 먹는 유저는 어떤 것을 선택해야 할지 모호함. "저녁"은 일반적으로 19시대로 인식함
- [+ 시간대 추가]를 두 번 눌러야 아침+저녁 두 스케줄이 설정되는데, 이 패턴을 처음에 발견하기 어려움 — 버튼을 한 번만 누르고 끝냈다고 생각할 수 있음
- **복용 조건이 약 단위(1개)로만 설정 가능** — 아침은 "식후", 저녁은 "식후"로 동일하면 OK지만, 시간대별로 조건이 다른 케이스를 처리할 수 없음 (현재 구조의 한계)

**잘 된 부분:** 이미 추가된 시간대는 바텀시트에서 비활성화되어 중복 등록 방지가 작동함

## 시나리오 3 — 오늘 복약 체크: 아침 드시모네 + 항우울제, 저녁 항우울제

### 실제 UI 구조 (MainScreen 코드 기반)

등록된 약이 있으면:
- 상단 헤더: "투약 관리" + "약 관리" 링크
- 내 약 요약 배너 (약 이름 2개 + 외 N개, 탭하면 약 관리로)
- "오늘의 복약" 섹션: N/M 완료 배지 + 날짜 + 진행률 바
- 시간대별 그룹(아침/점심/저녁/취침 전): CheckCard 목록
- 다음 예정 섹션 (주 1회 약이 오늘 아닌 날)
- 달력 뷰 (CalendarView)
- 주간 요약 카드

### CheckCard 구조

각 카드: 왼쪽 색상 도트 + 이름 + 메타정보(용량·조건·시간) + 오른쪽 체크 버튼  
체크 시: 카드 배경 연초록, 체크마크 표시  
미복용 배지: 예정 시간 30분 초과 미체크 시 빨간 "미복용" 뱃지

### 페르소나 A의 아침 체크

**동작:** 앱 열기 → 아침 그룹에 드시모네, 항우울제(아침) 두 카드가 보임 → 오른쪽 체크 버튼 탭 → 체크 완료

**잘 된 부분:**
- 시간대별 그룹핑(아침/저녁)으로 스크롤 없이 아침 약 한눈에 확인 가능
- 진행률 바(progress bar)가 완료도를 직관적으로 표시
- 체크 버튼이 카드 오른쪽에 독립적으로 위치 — 잘못 체크 방지

**혼란 포인트:**
- **체크 버튼이 작아 보일 수 있음** — 코드에 별도 크기 지정이 있지만, 카드 전체를 탭하는 것인지 버튼만 탭하는 것인지 헷갈릴 수 있음. 실제로는 버튼(`TouchableOpacity`)만 탭 가능, 카드 전체는 탭 불가
- **카드 전체 탭이 안 됨** — 기획서(8.1절)는 "약물 카드를 탭하면 체크" 동작을 명시하지만, 코드는 카드 내 버튼만 탭 처리. 카드 전체가 탭 영역이었으면 더 쉬웠을 것
- 항우울제 아침/저녁 카드가 같은 약인데 색상 도트만으로 구분 → "아침"/"저녁" 레이블이 메타 정보에 있어야 하는데 코드에서 `item.timeDetail`(시간)만 보여줌. 같은 이름의 항목이 두 개 보이면 혼란
  - 실제 코드: `{item.dosage || "용량 없음"} · {CONDITION_LABEL[item.condition]} · {item.timeDetail}` — 시간대 레이블(`아침`/`저녁`)이 카드 메타에 없고 섹션 헤더로만 표시됨. 아침 섹션에 항우울제, 저녁 섹션에 항우울제가 각각 보이는 구조는 맞지만, 바텀시트에서는 그룹 없이 단순 나열됨

### 페르소나 A의 저녁 체크 (22시)

**동작:** 저녁에 앱 열기 → 아침 약들은 체크됨 → 저녁/취침 전 그룹에 항우울제 카드 확인

**혼란 포인트:**
- 항우울제 복용 시간이 22시이면 "취침 전" 그룹에 배치됨. 저녁 섹션에 없어서 "왜 저녁 약이 안 보이지?" 의문 발생 가능
- **19:30이 지나면 항우울제(저녁/취침 전) 카드에 빨간 "미복용" 배지가 붙음** — 실제 복용 시간이 22시인 유저 입장에서 19:30부터 빨간 배지가 보이면 불필요한 불안감 야기. isOverdue 로직이 `timeDetail`(22:00 기준)이므로 22:30이 지나야 미복용 배지가 뜨는 게 맞음. 이건 올바르게 동작함

## 시나리오 4 — 달력 확인: 지난 2주 복약 기록

### 실제 UI 구조 (CalendarView 코드 기반)

- 월 단위 달력 (이전/다음 월 이동 화살표)
- 날짜 셀: 날짜 숫자 + 상태에 따른 배경색 원 + 하단 도트 행(약별 색상, 최대 3개)
- 상태별 원 색상: 완료=초록, 일부=노랑(`calendarDayPartial`), 미복용=빨강(`calendarDayMissed`), 미래=비활성
- 날짜 탭 → DateDetailSheet(바텀시트)

**페르소나 A의 달력 탐색 동작:**

스크롤해서 달력 섹션 도달 → 지난 2주 날짜를 하나씩 탭하여 확인 → 바텀시트로 상세 확인

**잘 된 부분:**
- 날짜별 색상(초록/노랑/빨강)으로 한눈에 복약 이력 파악 가능
- 도트가 약별로 색상이 달라 어떤 약을 복용했는지 시각 구분 가능
- 바텀시트에서 해당 날 체크/미체크 수정 가능

**혼란 포인트:**
- **달력이 메인 화면 하단에 묻혀 있음** — 유저는 스크롤해야 달력을 볼 수 있음. "체크리스트 → 다음 예정 → 달력" 순서라 달력 접근에 스크롤이 꽤 필요
- 달력이 스크롤 내부에 있고 별도 탭/뷰 없음 → 이력 확인 목적으로 바로 달력을 보고 싶을 때 직접 네비게이션 없음
- **도트가 최대 3개로 제한됨** (`getDayDots`의 `slice(0, 3)`) — 현재 3개 약이므로 OK이지만, 약이 4개 이상이면 일부 약은 달력에서 안 보임. 이 페르소나는 해당 없음
- 달력 하단에 **주간 요약 카드가 달력 아래에 있음** (코드: CalendarView 다음에 weekSummaryCard 렌더) — 기획서(8.4절)는 달력 위에 배치를 명시하나 코드는 달력 아래. 이 순서 역전은 주간 요약 접근성 저하

## 시나리오 5 — 미복용 대응: 어제 저녁 항우울제 빼먹음을 오늘 발견

### 실제 UI 구조 및 동작 메커니즘

`toggleCheck` 함수는 `date` 파라미터를 받으며, 오늘 날짜가 기본값이지만 **달력 → 바텀시트 경로를 통해 과거 날짜의 체크도 가능**함.

경로: 달력에서 어제 날짜 탭 → DateDetailSheet 오픈 → 어제의 항우울제(저녁) 카드에서 체크 버튼 탭 → 로그 생성됨

### 페르소나 A의 행동

아침에 앱 열기 → 어제 빼먹었다는 걸 인지 → 달력으로 스크롤 → 어제 날짜(노랑 또는 빨강 배경) 탭 → 바텀시트에서 항우울제(저녁) 체크

**잘 된 부분:**
- 과거 날짜 바텀시트에서 실제로 체크 처리가 가능한 구조 (toggleCheck에 date 전달됨)
- 어제 날짜가 일부 완료(노랑) 또는 전체 미복용(빨강)으로 색상 표시되어 즉시 인지 가능

**혼란 포인트 — 이것이 가장 큰 문제:**
- **메인 화면에 "어제 빠뜨린 약 알림"이 없음** — 기획서(시나리오 4)는 "다음 날 투약 탭 진입 시 '어제 우울증약(저녁)을 놓쳤어요' 인앱 알림 표시"를 명시하지만 코드에 구현 없음
- 유저가 스스로 달력을 보러 스크롤하지 않으면 어제 누락을 인지할 방법이 없음
- "달력에서 어제 탭 → 바텀시트" 경로 자체를 유저가 알아야 함 — 이 페르소나(복잡한 UI 싫어함)는 달력을 탭하면 과거 수정이 가능하다는 것을 처음에 모를 수 있음
- 바텀시트에서 체크해도 시각적으로 "사후 체크"인지 "당일 체크"인지 구분 없음 — 어제 22시에 먹은 것처럼 기록되는 게 맞는지 불분명
- **체크 해제(실수 방지 다이얼로그)**: 기획서 8.1절은 체크 해제 시 확인 다이얼로그가 있어야 한다고 하지만, 코드는 `toggleCheck`에서 바로 `pending`으로 변경됨 — 다이얼로그 없음
- **"미복용"으로 명시적 기록하는 기능 없음** — 현재는 체크하거나 안 하거나 두 가지만. "어제 못 먹었고 그냥 skip으로 처리"하는 flow 없음

## UX 문제점 우선순위 리스트

---

### 심각도: 높음

**[H1] 어제 누락 약 인앱 알림 없음**
- 기획서 명시 기능이지만 코드 미구현
- 이 페르소나의 핵심 pain point("병원 다음 날 약 빼먹는 경우 잦음")를 해결하는 핵심 기능
- 현재 유저가 스스로 달력 스크롤 → 어제 날짜 탭 → 바텀시트 열기까지 해야 인지 가능. 행동 장벽 극히 높음
- **개선:** 탭 진입 시 최상단에 "어제 N개 약을 놓쳤어요" 배너 노출

**[H2] 카드 전체 탭 불가 — 체크 버튼만 탭 가능**
- 기획서 8.1절은 "약물 카드를 탭하면 체크" 명시
- 실제 코드: `CheckCard` 내 `TouchableOpacity` 버튼만 탭 영역. 카드 자체는 탭 불가
- 스마트폰 사용자 직관(카드 전체 = 탭 가능)과 불일치 → 처음에 카드를 눌렀는데 반응이 없어 당황
- **개선:** CheckCard의 View를 TouchableOpacity로 교체, 버튼 중복 제거 또는 카드 탭 = 체크 처리

**[H3] 약 등록 후 연속 등록 흐름 없음**
- 저장 후 `setScreen("main")`으로 바로 메인 이동
- 3개 약 등록 시 매번 메인 → "약 관리" → "+ 새 약 등록하기" 경로 반복 필요
- 기획서는 "다른 약도 등록할까요?" 프롬프트를 명시하나 미구현
- **개선:** 저장 직후 "등록 완료. 다른 약도 추가할까요? [계속 추가] [완료]" 다이얼로그 추가

---

### 심각도: 중간

**[M1] "요일 지정" 레이블이 "주 1회"와 다른 멘탈 모델**
- 위고비 같은 주 1회 주사 약 등록 시 "요일 지정" 버튼 선택해야 함
- 복용 주기가 "매일" / "요일 지정"이라는 라벨은 "얼마나 자주"(주기)가 아닌 "어떤 요일에"(스케줄)로 읽힘
- **개선:** "매일" / "특정 요일" 또는 "요일 선택"으로 레이블 변경. 또는 "주 1회(요일 지정)"로 구체화

**[M2] 항우울제 아침/저녁 카드가 같은 이름으로 두 번 등장**
- 체크리스트에서 "항우울제 / 용량 · 식후 · 08:00"와 "항우울제 / 용량 · 식후 · 22:00"가 나란히 보임
- 카드 내 메타 정보에 시간대 레이블(아침/취침 전)이 없고 섹션 헤더로만 구분
- 스크롤 중이거나 빠르게 보면 같은 약인데 왜 두 개인지 혼란
- **개선:** 카드 이름 옆 또는 메타에 "(아침)", "(저녁)" 시간대 레이블 병기

**[M3] 달력이 메인 하단에 묻혀 있고 주간 요약이 달력 아래에 위치**
- 달력 접근에 스크롤 여러 번 필요 → 빠르게 이력 확인하려는 유저 이탈
- 주간 요약이 달력 아래에 있어 먼저 달력을 지나쳐야 보임 (기획서는 달력 위 명시)
- **개선:** 달력을 별도 섹션 탭으로 분리하거나, 주간 요약을 달력 위로 이동

**[M4] 체크 해제 시 확인 다이얼로그 없음**
- 기획서 8.1절: 체크된 카드 다시 탭 → "실수로 체크했어요?" 확인 다이얼로그 명시
- 코드: 바로 `pending` 상태로 변경, Alert 없음
- 실수로 체크 해제 시 복구 수단 없음
- **개선:** 체크 해제 시 `Alert.alert("체크를 해제할까요?", "", [취소, 확인])` 추가

**[M5] "취침 전"(22:00)과 "저녁"(19:00) 시간대 구분이 모호**
- 이 페르소나의 항우울제 복용 시간은 22시 → 어떤 시간대 선택해야 할지 불명확
- "저녁"을 선택하면 기본값 19:00이 붙음 → 실제 복용 시간과 3시간 차이 → 미복용 배지 오발령 가능성
- 직접 입력(스테퍼)을 써야 하는데 이 경로를 바텀시트에서 발견해야 함
- **개선:** 바텀시트에서 시간대 옵션에 예시 시간 명기 강화 또는 프리셋 직접 수정 허용

---

### 심각도: 낮음

**[L1] 약 관리 화면에서 편집 기능 없음 — 삭제만 가능**
- ManageScreen 코드: 약 카드에 [삭제] 버튼만 있고 편집 기능 없음
- 기획서 와이어프레임은 [편집] 버튼을 명시
- 용량 변경, 시간 수정 등 편집 필요 시 삭제 후 재등록해야 함 → UX 매우 불편
- **개선:** 약 카드에 [편집] 버튼 추가 및 EditScreen 구현

**[L2] 달력 도트가 완료된 것만 표시 (미완료 도트 없음)**
- `getDayDots` 함수: `log.status === "taken"`인 것만 도트 표시
- 기획서(8.3절)는 "채워진 도트 = 복용, 빈 도트(테두리) = 미복용" 명시
- 실제 코드는 미복용 도트를 그리지 않음 → 어떤 약이 예정되었는지 달력에서 불명확
- **개선:** 예정 약(오늘 이전, 복용 대상)의 미복용 항목도 빈 도트로 표시

**[L3] 주간 요약에 놓친 약 상세 없음**
- `getWeekSummary` 함수: `rate`(복약률)과 `missedCount`(총 횟수)만 반환
- 기획서(8.4절)는 "놓친 약: 우울증약-저녁 (수요일)" 같은 상세 정보 표시 명시
- 현재는 몇 회 놓쳤는지만 보여주고 어떤 약인지는 모름
- **개선:** missedItems 배열에 약 이름 + 시간대 + 날짜 포함하여 UI에 표시

**[L4] 데이터 비영구성 (로컬 state만, 앱 재시작 시 초기화)**
- `useState`로만 관리, 로컬 스토리지나 API 연동 없음 (createInjection API 호출은 있지만 로드 없음)
- 앱 재시작 시 모든 약 등록과 복약 기록이 사라짐
- 이 페르소나처럼 장기 복약 추적이 목적인 유저에게 치명적
- **개선:** AsyncStorage 또는 서버 API로 데이터 영속성 확보 (현재 MVP 단계이므로 우선순위 낮게 분류했으나 실제로는 런칭 전 필수)

**[L5] "다음 예정" 섹션의 용량/형태 정보 부족**
- `nextCard`에는 약 이름 + "N요일 · M일 후"만 표시
- 위고비처럼 주사제인 경우 "💉 피하주사" 등 형태 정보가 없음
- 기획서 와이어프레임은 "💉 위고비 0.5mg 주사 — 토요일(3일 후)" 형식 명시
- **개선:** 다음 예정 카드에 FORM_LABEL과 용량 추가

## 종합 평가

### 잘 구현된 부분

- Empty State → 체크리스트 → 달력 전환이 논리적
- 시간대별 그룹핑(아침/점심/저녁/취침 전)으로 복수 약물을 깔끔하게 정리
- 주 1회 약 "다음 예정" 카운트다운이 직관적
- 달력 날짜 색상(초록/노랑/빨강)으로 복약 현황 한눈에 파악
- 복수 시간대 등록 지원(같은 약의 아침+저녁)으로 기획서 핵심 요구사항 충족
- 미복용 배지(30분 초과)가 실시간 경고 제공

### 핵심 격차

기획서 v1.1이 목표한 바와 현재 코드 간의 가장 큰 격차는 세 가지다.

1. **능동적 알림 없음** — 놓친 약을 유저가 스스로 찾아야 함. 이 페르소나처럼 병원 다음 날 약을 자주 빼먹는 유저에게 가장 필요한 기능이 미구현
2. **카드 탭 UX 불일치** — 카드 전체가 아닌 작은 버튼만 탭 가능한 구조가 모바일 직관에 어긋남
3. **편집 기능 부재** — 약 용량/시간 변경 시 삭제 후 재등록 필요. 장기 사용자 이탈 원인

이 세 가지를 해결하면 페르소나 A의 일상 사용 만족도가 크게 올라갈 것으로 판단된다.
