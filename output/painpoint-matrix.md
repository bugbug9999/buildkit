## 0. 요약

1차·2차 사용자 테스트 4명(30대 직장인, 55세 주부, 34세 IT 파워유저, 46세 자영업 중장년)과 경쟁사 10+ 앱 리뷰 분석 결과, **GLP-1 기록 앱 사용자의 페인포인트는 3개 카테고리에 집중**된다.

1. **신뢰성 기초(trust baseline)** — 데이터가 저장되는가, 알림이 오는가, 재진입 시 기록이 살아있는가. 이 레이어가 깨지면 다른 모든 가치가 무너진다. 현재 GLP-Care의 최대 리스크.
2. **인지 부담(cognitive load)** — "지금 몇 mg인지", "이번 주 맞았는지", "어제 뭐 놓쳤는지". GLP-1의 증량 스케줄과 다중 복약 특성에서 반복 발생.
3. **자기 효능감(efficacy feedback)** — "내가 하고 있는 게 효과 있나?" 체중/칼로리/복약이 연결되어 변화를 보여주는 것. 경쟁사 공통 약점이자 GLP-Care 최대 기회.

이 문서는 4명의 테스트와 경쟁사 리뷰에서 반복 등장한 페인포인트 18개를 도출하고, 각 항목에 대해 경쟁사 대응 / GLP-Care 현재 / GLP-Care 향후 / 우선순위를 매핑한다.

## 1. 페인포인트 도출 근거

각 페인포인트는 **2명 이상의 페르소나에서 공통 발생**했거나 **경쟁사 리뷰에서 반복**되어야 목록에 올렸다. 단일 페르소나 고유 이슈는 제외했다 (예: 파워유저 A의 "요일 지정" 레이블 선호도).

- **페르소나 A** (30대 직장인, GLP-1 파워유저, 기능 완성도 관점)
- **페르소나 B** (55세 주부, 경구약, 접근성 관점)
- **페르소나 C / 미경** (34세 IT 직장인, 자가구매 위고비, 심미성·차별화 관점)
- **페르소나 D / 정호** (46세 자영업, 당뇨 전단계 다중 복약, 중장년 접근성·가족 관점)

## 2. 페인포인트 매트릭스

### 카테고리 A — 신뢰성 기초 (Trust Baseline)

#### P-01. 데이터 비영구성: 앱 재시작 시 모든 기록 초기화

**페인 강도:** 치명적
**발견:** 페르소나 A (L1) · C (최상위 불만) · D (이탈 결정적 순간)
**유저 보이스:**
- 미경: "앱을 껐다가 다시 켜도 내 데이터가 남아있어야 합니다. 이건 기본입니다."
- 정호: "아이고, 앱을 닫았다 열면 약이 다 사라져? 차라리 종이에 쓰지."

**경쟁사 해결:**
- Medisafe, MyTherapy, CareClinic 모두 로컬 DB + 클라우드 동기화 기본. MyTherapy는 무료인데도 cross-device sync 제공.
- MyFitnessPal은 계정 기반 서버 저장이 오히려 기본. 로컬 캐시는 보조.
- **이건 경쟁사에서 "불만"조차 아닌 기본 가정**. GLP-Care만의 결함.

**GLP-Care 현재:** useState만 사용. AsyncStorage / SQLite 미연동. 앱 종료 = 데이터 손실.

**GLP-Care 향후:**
- **즉시(Phase 0):** AsyncStorage (RN 기본) 또는 `expo-sqlite`로 로컬 영속화. 2~3일 작업.
- **Phase 1:** 계정 시스템 + 서버 동기화 (Supabase / Firebase). 기기 교체, 폰 분실 대비.
- **Phase 2:** iCloud / Google Drive 백업 연동 (미경 같은 iPhone 파워유저).

**우선순위:** 🔴 **P0 (출시 전 반드시)** — 이 한 가지 이슈로 두 페르소나의 3개월 이탈률이 각각 85%, 100%로 예측됨. 다른 어떤 기능보다 선행.

---

#### P-02. 푸시 알림 미구현: 저녁 약 놓칠 때 앱이 침묵

**페인 강도:** 높음
**발견:** 페르소나 A (누락 알림) · B (H5) · C (기대했는데 없음) · D (1순위 개선 요청)
**유저 보이스:**
- 정호: "식당이 바쁘면 저녁 약을 자꾸 빼먹어. 앱이 '야, 저녁 약 먹을 시간이다'고라도 알려줘야지."
- 기획서(v1.1) 명시 기능 — Phase 1 MVP 격상. 그러나 코드는 0줄.

**경쟁사 해결:**
- **Medisafe:** 핵심 경쟁력. 시간대별 리마인더 + 미복용 시 재알림 + "Medfriend" 보호자 알림.
- **MyTherapy:** 기본 푸시 + "사라진 약" 추적.
- **CareClinic:** 리마인더 기본. 단 무료는 약 1개 제한.
- **결론:** 복약 앱에서 푸시 알림은 "차별화"가 아니라 "참가비". 없으면 카테고리 진입 자체가 안 됨.

**GLP-Care 현재:** 알림 로직 전무. 미복용 30분 후 푸시(기획서 시나리오 4) 미구현.

**GLP-Care 향후:**
- **즉시:** `expo-notifications`로 기본 로컬 알림. 약별 시간대별 스케줄링. 2~3일 작업.
- **Phase 1:** 미복용 후 30분 재알림 + "어제 N개 놓쳤어요" 인앱 배너 (C-1 해결).
- **Phase 2:** 지능형 알림 억제 (이미 체크한 날은 울리지 않음, 패턴 학습).

**우선순위:** 🔴 **P0** — 복약 앱 카테고리 최소요건. 출시 전 필수.

---

#### P-03. 어제 놓친 약 인앱 배너 부재

**페인 강도:** 높음
**발견:** 페르소나 A (H1) · B (H5) · D (시나리오 6에서 발견)
**유저 보이스:** 앱 진입 시 "어제 내가 뭘 놓쳤지?"를 알려주지 않음. 유저가 달력을 스크롤해서 직접 확인해야 함.

**경쟁사 해결:**
- Medisafe, MyTherapy 모두 메인 진입 시 "missed dose" 카드 노출.

**GLP-Care 현재:** 기획서 명시, 코드 미구현. `missedCheck` 로직 없음.

**GLP-Care 향후:** MainScreen 상단 조건부 배너. `getCheckItemsForDate(yesterday)` 기반. 1일 작업.

**우선순위:** 🔴 **P0** — P-02와 함께 출시 전 처리.

### 카테고리 B — 인지 부담 (Cognitive Load)

#### P-04. 등록 단계 과다: 10단계 플로우

**페인 강도:** 높음
**발견:** 페르소나 B (H7, 3단계 기대 vs 10단계 현실) · D (중간에 멈칫거림) · A (파워유저 기준에서도 길다고 느낌)
**유저 보이스:**
- 정호: "약 하나 등록하는 데 이렇게 많이 눌러야 해? 세 개 다 하면 30번 이상이네."

**경쟁사 해결:**
- **Medisafe:** 처방전 사진 OCR로 자동 파싱. 3~4단계로 단축.
- **MyTherapy:** 약물 DB 자동완성 + 사진 인식.
- **CareClinic:** 빠른 추가 모드 (이름 + 시간만).

**GLP-Care 현재:** 이름 → 형태 → 용량 → 빈도 → 요일/매일 → 시간대 → 복용조건 → ... 단계 과다.

**GLP-Care 향후:**
- **Phase 1:** GLP-1 4개 약물(위고비/오젬픽/삭센다/리벨서스) **프리셋 템플릿**. 탭 2번에 등록 완료. → 타겟 유저의 80%를 3단계로 해결.
- **Phase 2:** 그 외 일반 약물 사전 DB (식약처 API 연동). 자동완성.
- **Phase 3:** 처방전 사진 OCR (Google ML Kit).

**우선순위:** 🟡 **P1** — GLP-1 프리셋은 난이도 낮음 + 차별화 큼.

---

#### P-05. 연속 등록 흐름 부재

**페인 강도:** 중간
**발견:** 페르소나 A (H3) · B (H7) · C · D (약 3개 등록 시 매번 메인 복귀)
**유저 보이스:** 약 하나 저장 후 메인 화면으로 튕겨 나감. 다음 약 등록 경로를 다시 찾아야 함. 정호는 3개 약 등록 중 두 번째에서 포기할 뻔함.

**경쟁사 해결:** Medisafe, MyTherapy 모두 "다른 약도 추가할까요?" 프롬프트 기본.

**GLP-Care 현재:** 저장 → 메인 이동.

**GLP-Care 향후:** 저장 후 "다른 약도 등록할까요? [예] [나중에]" Alert. 1시간 작업.

**우선순위:** 🟡 **P1**

#### P-06. GLP-1 증량 이력 시각화 부재 — "지금 몇 mg인가?"

**페인 강도:** 중간~높음 (GLP-1 사용자 한정, 타겟 유저의 60%)
**발견:** 기획서 시나리오 6에서 식별, 페르소나 A 우려, 미경(자가구매 위고비) 반복 언급
**유저 보이스:**
- 미경: "의사 없이 내가 용량 관리하는데, 지금 0.5mg이던가 1.5mg이던가 헷갈려."
- 기획서 v1.1의 DosageHistory 테이블이 직접 해결하려 한 문제.

**경쟁사 해결:**
- **Shed, Found:** 의료진이 관리하므로 유저가 직접 증량 시점을 알 필요 없음. 역으로 "자율적 기록"이 불가능.
- **Medisafe:** 용량 변경 시 과거 기록 소급 변경됨. GLP-1 증량 맥락 이해 없음.
- **PASTA:** 혈당 중심, 약물 용량 이력은 보조.
- **결론:** 이 페인포인트는 **경쟁사 모두 공백**. GLP-Care의 진짜 차별화 포인트 후보.

**GLP-Care 현재:** 기획서 v1.1에 DosageHistory 모델 + 달력 톤 변화 설계됨. 코드 구현 여부 미확인.

**GLP-Care 향후:**
- **Phase 1:** DosageHistory 모델 구현. 용량 변경 시점을 달력에서 색상 톤으로 구분.
- **Phase 2:** "현재 0.5mg 4주차 → 1.5mg 전환 예정일 D-7" 같은 증량 알림 (유저가 직접 등록한 스케줄 기반, 의료 조언 아님).
- **Phase 3:** GLP-1 표준 증량 프로토콜 템플릿 ("위고비 표준 16주 증량"). 유저 선택으로 자동 스케줄링.

**우선순위:** 🟢 **P2 (차별화 무기)** — P0 해결 후 가장 먼저 손대야 할 기능.

#### P-07. 다중 약물 하루 2회 복용 표현 모호

**페인 강도:** 중간
**발견:** 페르소나 A (A-2, 우울증약 아침/저녁 카드가 같은 이름으로 두 번) · 정호 (메트포르민 하루 2회)
**유저 보이스:** "메트포르민 카드가 두 개 있는데 둘 다 똑같이 보여서 어느 게 아침, 어느 게 저녁인지 헷갈려."

**경쟁사 해결:** Medisafe, MyTherapy는 시간대 레이블을 카드 메타에 표기.

**GLP-Care 현재:** v1.1에서 MedicationSchedule 분리로 데이터 모델은 해결. UI 레이블은 미흡.

**GLP-Care 향후:** CheckCard 메타 텍스트에 "아침 / 저녁" 명시. 4시간 작업.

**우선순위:** 🟡 **P1**

### 카테고리 C — 접근성 (Accessibility)

#### P-08. 폰트 12px — 중장년·노안 사용자 판독 불가

**페인 강도:** 높음 (GLP-1 사용자 중 40~60대 비율 고려 시)
**발견:** 페르소나 B (H1) · 정호 (3순위 요청)
**유저 보이스:**
- 정호: "약 이름은 괜찮은데, 용량이랑 시간이 너무 작아서 안경 벗고 봐도 잘 안 보여."
- 55세 주부 B: "복약 조건 글자가 안 보여서 식전인지 식후인지 모르겠어요."

**경쟁사 해결:**
- Medisafe, MyTherapy: 14~16px 기본, iOS Dynamic Type 지원.
- 삼성 헬스: 돋보기 모드 내장.

**GLP-Care 현재:** checkMeta fontSize 12, SUBTEXT 컬러(대비도 낮음).

**GLP-Care 향후:**
- **즉시:** fontSize 12 → 14. 텍스트 컬러 대비 WCAG AA 이상. 30분 작업.
- **Phase 1:** iOS Dynamic Type / Android font scale 대응.
- **Phase 2:** "큰 글씨 모드" 설정 옵션 (중장년 모드).

**우선순위:** 🔴 **P0** — 코드 1줄. 출시 전 반드시.

#### P-09. 터치 영역 44pt 미달

**페인 강도:** 높음
**발견:** 페르소나 B (H3 계열) · 정호 · 페르소나 C (스타일 이슈로 인지)
**유저 보이스:** 체크 버튼이 작아 손가락 가장자리가 옆 버튼에 닿기도 함.

**경쟁사 해결:** iOS HIG, Material Design 모두 44pt/48dp 최소. 경쟁사는 지키고 있음.

**GLP-Care 현재:** 체크 버튼 32×32.

**GLP-Care 향후:** 44×44 변경. 1시간 작업.

**우선순위:** 🔴 **P0**

#### P-10. 달력 도트 5px — 복약 상태 구분 불가

**페인 강도:** 중간~높음
**발견:** 페르소나 B (H-2) · C (스크린샷 공유 욕구 저하) · 정호
**GLP-Care 현재:** 도트 5px, 최대 3개 제한(slice(0,3)).

**GLP-Care 향후:** 8px 이상, 다중 약물 시 숫자 뱃지 또는 세로 스택. 2~4시간 작업.

**우선순위:** 🟡 **P1**

#### P-11. 달력이 화면 하단에 묻힘 (스크롤 6~7회)

**페인 강도:** 중간
**발견:** 페르소나 A (M3) · B (M-2) · C (M4) · D (스크롤 피로)
**유저 보이스:** 미경은 "패턴을 보고 싶은데 스크롤이 너무 많아서 포기"함.

**경쟁사 해결:** Medisafe, MyTherapy는 탭 전환 방식(체크 탭 / 달력 탭 분리). 또는 최상단 고정.

**GLP-Care 현재:** 단일 스크롤 뷰 하단에 달력 배치.

**GLP-Care 향후:**
- **Phase 1:** 투약 탭 내부에 세그먼트 컨트롤 ("오늘 / 달력"). 즉시 접근.
- 대안: 상단에 "이번 주 미니 스트립" + 하단 월간 달력.

**우선순위:** 🟡 **P1**

### 카테고리 D — 자기 효능감 (Efficacy Feedback)

#### P-12. 약 + 칼로리 연동 인사이트 부재 — "이 약이 효과 있나?"

**페인 강도:** 높음 (GLP-Care 차별화 핵심)
**발견:** 페르소나 A · C (2순위 요청) · 경쟁사 조사의 인사이트 1
**유저 보이스:**
- 미경: "위고비 맞은 날부터 일주일간의 칼로리 변화를 보여주세요. 그래야 약값 80만원이 아깝지 않다는 게 증명돼요."

**경쟁사 해결:**
- Shed / Found / WW: 의료진이 주관. 개인 분석은 의사 레벨에서만.
- Medisafe / MyTherapy: 복약 ↔ 식사 연결 전무.
- MyFitnessPal / 팻시크릿: 식사만.
- 삼성 헬스: 데이터는 있으나 GLP-1 맥락 없음.
- **결론:** 이 영역은 **완전한 공백**. 범용 복약 앱은 칼로리 DB 없고, 칼로리 앱은 약물 데이터 없음. GLP-Care만의 "원앱" 장점.

**GLP-Care 현재:** 두 기능 따로 존재. 연결 대시보드 없음.

**GLP-Care 향후:**
- **Phase 1:** "위고비 맞은 날 vs 안 맞은 날 평균 칼로리" 비교 카드 (2~3주 데이터 필요).
- **Phase 2:** 주사 후 1일/3일/7일 식욕 곡선 시각화. 체중 데이터 연동.
- **Phase 3:** "이번 주 복약률 90% → 체중 -0.8kg → 약이 기여한 것으로 추정" 같은 상관관계 내러티브 (의료 조언이 아니라 유저 기록 요약).

**우선순위:** 🟢 **P2 (차별화 무기 #1)** — 출시 후 4~8주 내 도입.

#### P-13. 주간 요약이 숫자만, 서사 없음

**페인 강도:** 중간

**유저 보이스:**

**경쟁사 해결:**

- **CareClinic:** 증상-약물 상관 리포트.

**GLP-Care 현재:** `missedCount` 숫자만. 어떤 약 어떤 요일에 놓쳤는지 상세 없음.

**GLP-Care 향후:**

**우선순위:** 🟡 **P1 (Phase 1)** / 🟢 **P2 (Phase 2)** — PDF 내보내기는 중장년층 결제 전환 트리거로 강력.

**페인 강도:** 중간

**우선순위:** 🟡 **P1** — 하지만 "자체 DB 경쟁"은 함정. 연동 전략이 정답.

## 4. 경쟁사가 풀지 못한 페인포인트 — GLP-Care의 해자

경쟁사 매트릭스에서 **모두 공백인 4개**는 GLP-Care의 진짜 차별화 기회다.

1. **P-06 GLP-1 증량 이력 시각화** — Shed·Found는 의료진 주관으로 유저가 볼 필요 없다고 가정. Medisafe는 증량 맥락 이해 없음. 한국의 자가구매 유저(미경)에게 유일하게 맞는 해법은 GLP-Care만 가능.
2. **P-12 약 + 칼로리 상관 인사이트** — 복약 앱은 칼로리 DB 없음, 칼로리 앱은 약물 데이터 없음. 한 앱에서 둘을 다루는 곳이 사실상 GLP-Care뿐.
3. **P-13 진료용 PDF (비의료 리포트)** — MyTherapy만 유사. 한국어 + GLP-1 맥락 포함은 전무.
4. **한국 로컬라이제이션** — Shed·Found·Embla 모두 한국 미지원. PASTA·삼성헬스는 범용이라 GLP-1 맥락 없음. "한국 GLP-1 유저 전용 기록 앱"이라는 포지션 자체가 비어 있음.

이 4가지를 Phase 2에서 확보하면 GLP-Care는 "니치 1위"로 자리잡을 수 있다. P0·P1은 "참가비"고, P2가 "무기"다.

## 6. 주의: 의료 조언 경계 유지

본 매트릭스의 모든 "향후 해결책"은 **유저가 직접 등록한 데이터의 기록·시각화 범위 내**로 제한된다. 다음은 절대 하지 않는다:

- 약 추천, 용량 조언, 증량 가이드 자동 생성
- "이 약은 이 약과 상호작용합니다" 같은 임상 판단
- 증상 입력 → 약물 추천
- 의사 상담 대체

예외적으로 허용되는 표현 패턴: "유저가 등록한 스케줄 기준 D-7 알림", "유저가 직접 기록한 체중·칼로리의 주간 추이", "유저가 등록한 복약 달성률의 PDF 요약". 이 선을 지키면 규제 부담 없이 경쟁사(Shed·Found)의 복잡성을 피할 수 있다.
