## 5. 다음 스프린트 권장 수정 (P1 — 코드 수정 포인트)

### 5-1. 어제 누락 알림 배너 (C-1)
`injection.tsx` → `MainScreen` 상단에 조건부 배너 추가:
```tsx
const yesterdayMissed = getCheckItemsForDate(medications, logs, yesterday)
  .filter(item => !item.log || item.log.status !== 'taken');

{yesterdayMissed.length > 0 && (
  <TouchableOpacity style={styles.missedBanner} onPress={() => { setSelectedDate(yesterday); setShowDateSheet(true); }}>
    <Text style={styles.missedBannerText}>
      어제 {yesterdayMissed.length}개 약을 놓쳤어요 →
    </Text>
  </TouchableOpacity>
)}
```

### 5-2. CheckCard 전체 탭 (C-2)
`CheckCard` 컴포넌트에서 카드 전체를 `TouchableOpacity`로 감싸고, 내부 체크 버튼 제거:
```tsx
// Before: View로 감싸고 내부에 TouchableOpacity 버튼
// After:
<TouchableOpacity
  activeOpacity={0.85}
  onPress={() => onToggle(item.medicationId, item.scheduleId, date)}
  style={[styles.checkCard, taken ? styles.checkCardTaken : null]}
>
  ...
</TouchableOpacity>
```

### 5-3. 터치 영역 + 폰트 확대
```ts
// styles 수정
checkButton: { width: 44, height: 44, borderRadius: 22 },
checkMeta: { fontSize: 14, color: SUBTEXT },
calendarDot: { width: 8, height: 8, borderRadius: 4 },
```

### 5-4. 체크 해제 확인 다이얼로그 (C-5)
`toggleCheck` 함수에서 taken → pending 분기에 Alert 추가:
```ts
if (existing?.status === 'taken') {
  Alert.alert('체크 해제', '복약 기록을 취소할까요?', [
    { text: '아니요' },
    { text: '취소', onPress: () => setLogs(prev => prev.map(...)) }
  ]);
  return;
}
```

## 종합 결론

두 페르소나가 공통으로 지적한 **C-1~C-8은 UX의 근본적 결함**으로, 파워유저건 디지털 약자건 모두 영향받는다. 이 중 C-1(누락 알림), C-2(카드 탭), B-1(폰트), B-3(터치 영역) 4가지는 코드 수정 난이도가 낮고 UX 개선 효과가 즉각적이어서 **이번 스프린트 내 반드시 처리** 권장.

페르소나 B 고유 이슈(B-1~B-9)는 접근성 기준 미달 항목이 다수로, GLP-1 약물 복용자 중 40~60대 비율을 고려하면 중요도가 높다. 특히 **글씨 크기와 터치 영역은 코드 1줄 수정**으로 해결 가능한데 방치하고 있는 상태.
