## 시나리오 1: 첫 방문 — 빈 화면에서 무엇을 해야 할지 파악

### 동작
앱을 처음 열고 투약 탭을 누른다. 빈 화면이 뜨고 💊 이모지 아래 "복용 중인 약을 등록해 주세요"라는 텍스트와 "약 등록하기" 버튼이 보인다. 버튼을 누른다.

### 혼란 포인트
- **"투약 관리"라는 헤더 텍스트** (fontSize: 22) — 이 정도는 읽힘. 하지만 우측 상단의 "약 관리" 링크는 fontSize 14, 초록색 텍스트만으로 처리되어 있어 버튼인지 모를 수 있음. 탭 영역도 좁아서 터치 실패 가능.
- **빈 화면만 보이는 상태에서 "이 앱이 뭘 해주는 앱인지" 설명이 부족함.** 기획서(7.1 Empty State)에는 "이런 기능을 사용할 수 있어요" 안내 리스트가 있지만, 실제 코드에는 이 부분이 구현되지 않았음. emptyDescription 텍스트가 "오늘 복약 체크, 달력 기록, 주간 요약을 한 번에 관리할 수 있습니다"로만 되어 있음 — 55세 주부가 이 설명 한 줄로 앱 가치를 파악하기 어려움.
- **"약 관리" vs "약 등록하기" 두 진입점이 혼재함.** 헤더 우상단의 "약 관리" 텍스트 링크와 카드 중앙의 "약 등록하기" 녹색 버튼 — 둘의 차이를 알 수 없어 "어디를 눌러야 하지?" 혼란.

### 잘 된 부분
- 💊 이모지(fontSize 48)가 눈에 확 들어와 "약 관련 탭"임을 즉시 인식 가능.
- "약 등록하기" 버튼이 초록 배경 + 흰 글씨 + padding 13으로 충분히 눈에 띔.
- 화면이 단순해서 "일단 이 버튼 하나만 누르면 되겠다"는 판단은 가능.

## 시나리오 2: 약 등록 — 혈압약 + 당뇨약 + 비타민D 3가지

### 동작
"약 등록하기" 또는 "약 관리" → "새 약 등록하기"를 눌러 등록 화면 진입. 총 3개 약을 순서대로 등록 시도.

**약 1 — 혈압약(암로디핀): 아침 1정, 매일**

"약 이름" 필드에 "혈압약" 또는 "암로디핀" 입력. "복용 형태"에서 💊 경구 선택. "용량"에 "1정" 입력. "복용 주기"에서 "매일" 선택. "복용 시간대"에서 "+ 시간대 추가" 눌러 바텀시트 열기 → "빠른 선택"에서 "아침 08:00" 선택.

**약 2 — 당뇨약(메트포르민): 아침·저녁 매일**

같은 흐름으로 진입. 복용 시간대에서 "아침 08:00" 선택 후, 다시 "+ 시간대 추가"를 눌러 바텀시트를 열어 "저녁 19:00"도 선택.

**약 3 — 비타민D: 저녁 매일**

비타민D 이름 입력, 경구, 매일, 저녁 선택.

### 혼란 포인트

**[A] 등록 화면 진입 경로가 2가지라 혼란**  
- 빈 화면에서는 "약 등록하기" → `AddScreen`으로 바로 이동  
- 약이 1개 이상 등록된 후에는 "약 관리" → `ManageScreen` → "+ 새 약 등록하기" → `AddScreen`  
- 즉, 등록 경로가 단계마다 달라짐. 특히 2번째 약을 등록할 때 "아까는 바로 눌렀는데 지금은 어디 가야 하지?" 혼란이 발생함.

**[B] "복용 시간대" 필드가 직관적이지 않음 — 심각**  
코드를 보면 초기값 `draft.schedules = []`이라 화면에 아무것도 없고, "+ 시간대 추가" 텍스트 버튼만 있음 (fontSize 14, 초록 배경 없음 — 단순 텍스트 느낌). 55세 주부는 이 버튼을 못 보고 넘어가거나, "시간대가 뭔지" 몰라서 안 누를 수 있음.  
→ 버튼을 눌러야 바텀시트가 열리고, 거기서 "빠른 선택" vs "직접 입력" 두 섹션을 이해해야 함. 이미 2단계가 더 생김.

**[C] 당뇨약 아침+저녁 이중 등록이 어려움 — 심각**  
아침 추가 후 저녁을 추가하려면 "+ 시간대 추가"를 또 눌러야 함. 이 버튼 자체가 눈에 잘 안 띄는 `styles.addTimeButton` (border만 있는 흰 배경)으로 구현되어 있어, 이미 아침을 추가한 후 버튼을 다시 찾는 것이 어렵다.

**[D] "복용 조건" 용어 이해 불가 — 높음**  
"공복 / 식전 / 식후 / 무관" 중 "무관"이 뭔지 모름. 55세 주부 입장에서는 "그냥 아무 때나 먹어도 돼요?" 하는 질문이 생기는데 설명이 없음. 코드에서 `CONDITION_LABEL`은 한 단어 레이블만 있고 부연 설명이 없음.

**[E] 약 이름 필드 placeholder "예: 위고비, 혈압약, 유산균"**  
이 부분은 페르소나 B에게 유리함 — "혈압약"이라는 일상어도 예시로 있어서 영어 약품명 없이 등록 가능하다는 걸 알 수 있음.

**[F] 저장 버튼이 우상단 텍스트 "저장"으로만 존재**  
`headerSaveButton`이 화면 상단 우측에 작은 텍스트로만 있음 (fontSize 16이지만 넓은 터치 영역 없음). 55세 주부는 스크롤을 내리다가 "이제 다 됐는데 어떻게 저장하지?"라고 막힐 가능성 높음. 하단 큰 버튼이 없음.

**[G] 총 단계 수 — 심각**  
약 1개 등록 기준: 탭 진입 → 약 관리 → 새 약 등록 → 이름 입력 → 형태 선택 → 용량 입력 → 주기 선택 → 시간대 추가(바텀시트 열기) → 시간 선택 → 저장 = **최소 10단계**. 페르소나 B의 허용치 3단계를 4배 초과.

### 잘 된 부분
- 복용 형태 버튼(💊 경구 / 💉 주사 / 🧴 기타)이 세그먼트 형태로 눈에 잘 띔.
- "아침 08:00", "점심 12:00", "저녁 19:00", "취침 전 22:00" 빠른 선택 버튼 — 시간을 직접 입력하지 않아도 되어 편리함.
- 복용 주기 "매일" / "요일 지정" 2가지로 단순화됨.

## 시나리오 3: 오늘 복약 체크 — 아침 약 2개 체크

### 동작
메인 화면 진입 후 "오늘의 복약" 섹션에서 아침 그룹의 혈압약, 당뇨약(아침) 두 카드를 순서대로 우측의 원형 버튼을 눌러 체크.

### 혼란 포인트

**[A] 체크 버튼이 작음 — 높음**  
`checkButton`의 크기가 width 32, height 32. 최소 권장 터치 영역(44pt) 미달. 55세 주부의 손가락 정밀도로는 실수로 옆을 누를 수 있음. 특히 카드 우측 끝에 붙어 있어 엄지로 닿기 어려운 위치.

**[B] 체크 후 해제 시 확인 다이얼로그**  
기획서에는 "실수로 체크했어요?" 확인 다이얼로그가 명시되어 있지만, 실제 코드(`toggleCheck`)에는 즉시 토글만 구현됨 — Alert 없이 바로 상태 변경. 기획과 구현 불일치. 단, 이는 페르소나 B에게 오히려 유리할 수 있음(확인 단계 없어서 더 쉬움).

**[C] 복약 카드의 메타 정보 폰트 너무 작음 — 높음**  
`checkMeta` (용량·복용조건·시간 표시)의 fontSize가 12. 예: "1정 · 식후 · 08:00" — 55세 주부가 읽기 매우 어려운 크기. 약 이름(fontSize 15)과 메타(fontSize 12) 간 크기 차이가 있지만 메타 자체가 너무 작음.

**[D] 진행률 표시 바 높이 6px — 낮음**  
`progressTrack` 높이 6. 시각적으로 존재감이 없어 "얼마나 했나" 파악이 어려움. 하지만 상단의 "0/3 완료" 배지가 함께 있어 보완 가능.

### 잘 된 부분
- 체크 후 카드 배경이 흰색에서 연한 초록(`#D1FAE5`)으로 바뀌어 "됐다"는 피드백이 명확함.
- 약 이름 글씨가 fontSize 15, bold — 비교적 잘 읽힘.
- 시간대(아침/저녁)별로 그룹 헤더가 있어 "아침 약이 어디 있는지" 파악 쉬움.
- "N/N 완료" 배지가 연초록 배경으로 눈에 잘 띔.

## 시나리오 4: 달력 확인 — 지난 일주일 복약 기록

### 동작
메인 화면을 스크롤 내려 달력 영역을 찾는다. 날짜 셀을 탭해서 지난 날의 상세 기록을 본다.

### 혼란 포인트

**[A] 달력이 화면 하단에 있어 찾기 어려움 — 중간**  
메인 화면 레이아웃 순서: 헤더 → 내 약 요약 배너 → 오늘의 복약 → 시간대별 체크카드 → 다음 예정 → 달력 → 주간 요약. 달력이 상당히 아래 위치. 스크롤을 많이 내려야 보임. 55세 주부는 "달력 어디 있지?" 하며 헤매거나 그냥 포기할 수 있음.

**[B] 달력 날짜 셀 크기가 작음 — 높음**  
`calendarDayCircle`의 크기 32×32. `calendarCell`의 minHeight 48이지만 실제 터치 대상(원)은 32×32. 7칸을 화면 너비에 나눠야 하므로 구조적 한계. 55세 주부가 정확히 원하는 날짜 셀을 탭하기 어려움.

**[C] 달력 도트가 너무 작음 — 높음**  
`calendarDot`의 width/height 5px, marginHorizontal 1px. 약별 색상 도트가 5px 크기 — 55세 주부가 보이지 않을 가능성 높음. "초록점이 있는지 없는지"를 인식하는 것이 사실상 불가능한 크기.

**[D] 달력 색상 의미를 모름 — 중간**  
초록/노랑/빨강 배경색이 "전체완료/일부완료/미복용"을 의미하지만 범례가 없음. 기획서(7.2)에는 범례가 있지만 코드에는 구현되지 않음. 55세 주부는 색상 의미를 스스로 파악하기 어려움.

**[E] 달력 좌우 화살표(‹ ›) 인식 문제**  
`calendarArrowText`의 fontSize 22, color SUBTEXT(#64748B). 특수문자 ‹ ›가 이전/다음 달 이동 버튼인지 모를 수 있음. "이게 버튼이구나"라고 인식하는 것 자체가 어려움.

**[F] 날짜 탭 → 바텀시트 상세 내용**  
바텀시트에는 "N월 N일 요일" 헤더 + 체크 항목 목록이 표시됨. 이 구조 자체는 단순해서 혼란 가능성 낮음. 단, 바텀시트가 열리는 것 자체를 "뭔가 올라왔는데 어떻게 닫지?" 당황할 수 있음. 닫기 버튼이 별도로 없고 배경 터치로 닫힘 — 설명 없음.

### 잘 된 부분
- 날짜가 "일 월 화 수 목 금 토" 한국어로 표기됨.
- 완료된 날짜가 초록빛으로 구분되어 "이 주에 잘 먹었구나" 전체적 패턴은 파악 가능.
- 바텀시트 상단에 드래그 핸들(`sheetHandle`)이 있어 시각적 단서 제공.

## 시나리오 5: 미복용 대응 — 저녁 당뇨약 빼먹고 다음 날 발견

### 동작
저녁에 당뇨약(메트포르민)을 빼먹었지만 알림이 없어서 몰랐음. 다음 날 아침에 앱을 열어 발견.

### 혼란 포인트

**[A] 알림(푸시 노티) 구현 없음 — 심각 (기획 vs 구현 Gap)**  
기획서(8.2)에는 복용 예정 시간 30분 경과 시 푸시 알림이 가도록 명시되어 있음. 하지만 injection.tsx 코드 전체에 `Notifications`, `expo-notifications`, `PushNotification` 등 알림 관련 import나 코드가 전혀 없음. 즉, **페르소나 B는 저녁 당뇨약을 빠뜨려도 알림을 절대 받지 못함.** 이 시나리오 자체가 현재 구현에서는 작동하지 않음.

**[B] "미복용" 뱃지는 있으나 다음 날엔 보이지 않음 — 높음**  
`isOverdue()` 함수가 `dateStr !== today`면 `dateStr < today`만 확인해서 true를 반환하므로, 어제 날짜의 미복용 카드에는 "미복용" 뱃지가 표시됨. 하지만 메인 화면의 체크카드는 **오늘 날짜(`today`)** 기준으로만 렌더링됨. 즉, 다음 날 메인 화면을 열면 어제 빠뜨린 약에 대한 어떤 인앱 알림도, 경고 메시지도 없음.

**[C] "어제 N약을 놓쳤어요" 인앱 알림 없음 — 높음**  
기획서(4. 시나리오 4)에 "다음 날 투약 탭 진입 시 '어제 우울증약(저녁)을 놓쳤어요'라는 인앱 알림이 표시된다"고 명시되어 있지만 코드에 이 기능이 없음.

**[D] 달력으로 확인하는 유일한 방법 — 중간**  
다음 날 달력을 스크롤해서 어제 날짜를 탭하면 당뇨약(저녁)이 미체크 상태임을 바텀시트에서 볼 수 있음. 하지만 55세 주부가 "어제 안 먹었는지 확인하려면 달력을 찾아서 탭해야 한다"는 흐름을 스스로 발견하기 매우 어려움.

**[E] 주간 요약에 "놓친 횟수 N회"만 표시 — 중간**  
`weekSummaryMissedValue`로 빨간색 "N회"는 표시되지만, **어떤 약을 언제 놓쳤는지**는 표시 안 됨. 기획서(8.4)에는 "우울증약-저녁 (수요일)" 같이 구체적인 약 이름+시간대+요일이 나와야 한다고 했지만, 코드에는 count만 있음.

### 잘 된 부분
- 달력 셀의 배경색(노랑=일부 완료, 빨강=전체 미복용)으로 "이날 뭔가 빠뜨렸구나"를 시각적으로 알 수 있음 (단, 셀이 작아서 색상 구분이 어려운 건 앞서 언급).
- 주간 요약의 복약률 % 수치가 크게(fontSize 28) 표시되어 "이번 주 잘 했는지 못 했는지" 숫자로 파악 가능.

## 종합 평가 및 핵심 개선 제안

### 이 페르소나가 성공적으로 수행할 수 있는 것
- 첫 방문 후 "약 등록하기" 버튼 찾아 누르기 (O — 버튼이 크고 명확)
- 오늘 아침 약 2개 체크 (△ — 체크 버튼이 작지만 카드 자체는 인식 가능)
- 달력에서 오늘 날짜 확인 (△ — 파란 테두리로 구분되지만 전반적으로 작음)

### 이 페르소나가 실패할 가능성이 높은 것
- 당뇨약 아침·저녁 이중 등록 (X — "시간대 추가"를 두 번 해야 한다는 것을 모를 가능성 높음)
- 저녁 당뇨약 미복용 인지 (X — 알림 없음, 인앱 안내 없음)
- 달력에서 지난 주 복약 상세 확인 (X — 달력 위치 파악, 도트 인식, 셀 터치 모두 어려움)
- 빠뜨린 약 사후 확인 (X — 관련 피드백 기능 자체 미구현)

### 최우선 개선 3가지
1. **알림 구현** — 복용 시간 30분 경과 시 푸시 알림. 페르소나 B에게 가장 중요한 안전망.
2. **등록 단계 단순화** — 빈 화면 기준 최대 3~4단계로 압축. "약 이름 → 언제 먹어요? (아침/저녁 선택) → 등록" 3단계 플로우.
3. **체크 버튼 및 터치 영역 확대** — checkButton을 최소 44×44pt로, 가능하면 카드 전체를 터치 영역으로 확장.
