# GLP-Care 투약 관리 앱 사용자 테스트 보고서

**테스터**: 페르소나 A (30대 직장인, 위고비+드시모네+항우울제)
**테스트 코드**: injection.tsx

---

## 시나리오 1: 첫 진입 — 빈 화면

### 동작
앱을 열면 상단에 "투약 관리" 헤더와 우측 "약 관리" 텍스트 버튼이 보인다. 화면 중앙에 💊 아이콘과 "복용 중인 약을 등록해 주세요" 카드가 뜬다. 설명 텍스트 읽고 "약 등록하기" 버튼을 누른다.

### 혼란 포인트
- **헤더의 "약 관리" vs 카드의 "약 등록하기"**: 두 버튼이 다른 화면으로 간다는 게 직관적으로 보이지 않는다. "약 관리"(→ ManageScreen)와 "약 등록하기"(→ AddScreen)의 차이를 초기 사용자는 모른다. 둘 다 비슷하게 생겼는데 결과가 다르다.
- 빈 상태에서 상단 "약 관리"를 먼저 누르면 "등록된 약이 없습니다"라는 빈 목록이 나온다. 거기서 다시 "새 약 등록하기"를 눌러야 한다. 불필요한 단계가 하나 더 생긴다.

### 잘 된 부분
- Empty state 설명 문구("약을 등록하면 오늘 복약 체크, 달력 기록, 주간 요약을 한 번에 관리할 수 있습니다")가 기능 가치를 구체적으로 전달한다.
- "약 등록하기" CTA가 시각적으로 명확하고 초록색으로 주목도가 높다.

## 시나리오 2: 약 등록 — 3가지 약 순서대로 등록

### ① 위고비 (주 1회, 금요일, 주사)

**동작**  
AddScreen 진입. 약 이름 "위고비" 입력 → 복용 형태 "💉 주사" 선택 → 용량 "0.5mg" 입력 → 복용 주기 "요일 지정" 선택 → 요일 칩에서 "금" 탭 → 시간대 "+ 시간대 추가" → 모달에서 "아침 08:00" 빠른 선택 → 복용 조건 "무관" → 저장.

**혼란 포인트**  
- **시간 입력 흐름이 끊긴다**: "+ 시간대 추가"를 누르면 바텀시트 모달이 올라온다. 빠른 선택("아침 08:00")을 탭하면 모달이 닫히면서 스케줄 카드가 생긴다. 이 흐름은 이해 가능하지만, "아침 08:00" 프리셋이 이미 선택됐는데 시간을 바꾸고 싶을 때 어떻게 하는지 알 수가 없다 (편집 불가, 삭제 후 재등록해야 한다).
- 직접 시간 입력 스테퍼: 기본값이 08:00이고 위고비를 오전에 맞으면 그냥 프리셋 쓰면 되니까 이 케이스에선 괜찮다.

**잘 된 부분**  
- "요일 지정" 선택 시 요일 칩이 바로 나타나는 흐름이 자연스럽다.
- 복용 형태 아이콘(💉)이 주사라는 걸 한 번에 파악하게 해준다.

---

### ② 드시모네 (매일 아침, 식후)

**동작**  
약 이름 "드시모네" → 복용 형태 "💊 경구" → 용량 "1정" → 복용 주기 "매일" → "+ 시간대 추가" → "아침 08:00" 선택 → 복용 조건 "식후" → 저장.

**혼란 포인트**  
없음. 가장 단순한 케이스라 흐름이 매끄럽다.

**잘 된 부분**  
복용 조건 칩(공복/식전/식후/무관)이 한눈에 보이고 선택이 즉각 반영된다.

---

### ③ 항우울제 (매일 아침 + 저녁)

**동작**  
약 이름 "항우울제" → "💊 경구" → 용량 입력 → "매일" → "+ 시간대 추가" → "아침 08:00" 선택 → 모달 닫힘 → **다시 "+ 시간대 추가"** → "저녁 19:00" 선택 → 저장.

**혼란 포인트**  
- **핵심 문제**: "아침"을 추가하면 모달이 즉시 닫힌다. 저녁을 추가하려면 다시 "+ 시간대 추가"를 눌러야 한다는 게 전혀 안내되지 않는다. 이 과정에서 **"아, 저녁도 추가해야 하는데 어디서 하지?"** 라는 순간이 발생한다. 버튼 텍스트가 "+ 시간대 추가"로 동일하니 한 번 더 눌러야 한다는 걸 추론해야만 한다.
- 모달에서 "직접 입력" 섹션이 보이는데, 항우울제를 저녁 22시에 먹는 사람이라면 프리셋 "저녁 19:00"가 안 맞아서 스테퍼를 써야 한다. 스테퍼에서 시간을 19→22로 올리려면 ▲를 3번 눌러야 한다 — 이건 괜찮다. 그런데 분침이 0, 15, 30, 45만 선택 가능해서 정각(00분)은 가능하다.
- 등록 완료 후 아무런 성공 피드백 없이 바로 메인 화면으로 전환된다. "잘 저장됐나?" 확인이 안 된다.

**잘 된 부분**  
- 기술적으로는 아침+저녁 2개 스케줄이 하나의 약에 잘 붙는다. 이게 핵심 기능이고 실제로 동작한다.
- 저장 버튼이 비활성("저장" 회색)에서 입력이 완료되면 활성화되는 피드백이 있다.

## 시나리오 3: 오늘 복약 체크

### 동작 (아침)
메인 화면 진입. 상단에 "오늘의 복약", 진행률 바(0/3 완료). 아침 그룹에 드시모네, 항우울제(아침). 저녁 그룹에 항우울제(저녁). 드시모네 카드 탭 → 초록 배경 + ✓ 표시. 항우울제(아침) 탭 → 동일하게 체크. "2/3 완료" 배지 업데이트.

### 동작 (저녁)
저녁에 앱 열면 아침 두 개는 이미 체크됨. 항우울제(저녁) 카드만 미체크. 탭 → 체크.

### 혼란 포인트
- **체크 해제가 너무 쉽다**: 코드를 보면 `toggleCheck`은 taken → pending을 **확인 없이** 즉시 처리한다. 실수로 체크를 눌렀다가 다시 누르면 그냥 해제된다. 기획서에서는 "실수로 체크했어요?" 확인 다이얼로그를 명시했는데 구현되지 않았다.
- **체크 버튼 크기**: `checkButton`이 32×32px. 스마트폰 터치 최솟값인 44×44에 못 미친다. 급하게 아침에 빠르게 탭할 때 빗나갈 수 있다.
- **카드 전체 탭 vs 버튼 탭**: 카드 전체가 아니라 우측 원형 버튼만 탭 가능하다. 처음에 카드 본문을 눌렀다가 반응이 없어서 "어디를 눌러야 하지?" 할 수 있다.
- **복용 조건이 카드에 텍스트로만 표시**: "1정 · 식후 · 08:00"처럼 메타 정보가 작은 글씨(12px)로 한 줄에 압축돼 있다. 빠르게 훑을 때 식후인지 식전인지 안 보인다.

### 잘 된 부분
- 시간대별 그룹화(아침/저녁)가 명확해서 "지금 뭘 먹어야 하는지" 즉시 파악된다.
- 위고비 금요일 주사 → "다음 예정: 금요일 · N일 후" 카드가 자연스럽게 분리됐다. 오늘 체크 목록과 헷갈리지 않는다.
- 진행률 바 + N/M 완료 배지 조합이 직관적이다.
- 시간 초과 30분 후 "미복용" 빨간 뱃지가 나타나는 건 자연스러운 시각 경고다.

## 시나리오 4: 달력에서 지난 2주 복약 기록 확인

### 동작
메인 화면 스크롤 다운 → 달력 뷰 노출. 이번 달 날짜들이 배경색으로 구분됨. 날짜 탭 → DateDetailSheet 바텀시트 팝업.

### 혼란 포인트
- **도트가 없다**: `getDayDots`는 status가 "taken"인 로그만 기반으로 도트를 그린다. 등록 직후라면 아직 체크 기록이 없어서 모든 날짜 셀이 빨간 배경(missed)에 도트 없음이다. 2주 치 데이터가 쌓인 뒤에야 시각적으로 의미가 생긴다. **첫 사용 직후 달력이 온통 빨간색**으로 보이는 것은 사용자에게 매우 부정적인 첫인상이다.
- **달력 범례가 없다**: 초록/노랑/빨강 배경색의 의미가 화면 어디에도 설명되지 않는다 (기획서엔 범례를 넣도록 명시했지만 코드에 없다).
- **과거 날짜 탭 시 피드백**: 바텀시트가 올라올 때 "이 날짜를 탭하면 기록을 수정할 수 있다"는 안내가 없다. 단순 조회인지 수정 가능한지 알 수 없다.
- **달력과 주간 요약이 역순 배치**: 달력이 위에, 주간 요약 카드가 아래에 있다. 달력까지 스크롤해야 요약을 볼 수 있다. 주간 요약이 더 자주 보는 정보인데 아래에 있어서 매번 스크롤이 필요하다.

### 잘 된 부분
- 달력 월 이동 `‹ ›` 화살표가 명확하다.
- 날짜 탭 → 바텀시트 흐름이 자연스럽고 빠르다.
- 주간 복약률 % + 놓친 횟수 두 수치 조합이 핵심 정보를 압축한다.

## 시나리오 5: 어제 저녁 항우울제 미복용 발견 → 대응

### 동작
달력에서 어제 날짜 탭 → DateDetailSheet 오픈. "항우울제(저녁)" 카드에 빨간 "미복용" 뱃지 표시. 우측 빈 체크박스 탭 → 상태가 ✓로 바뀐다. 시트 닫으면 달력에서 어제 날짜 색이 빨강 → 초록으로 변경.

### 혼란 포인트
- **소급 체크 가능하다는 걸 모른다**: 과거 날짜 바텀시트에서 체크박스가 탭 가능하다는 어포던스가 없다. "조회 전용"처럼 보이는 화면에서 수정이 가능하다는 게 비직관적이다. 많은 사용자가 그냥 닫고 "이미 늦었네"하고 넘어갈 가능성이 높다.
- **소급 체크 시 기록 시각**: `checkedAt: new Date().toISOString()`으로 지금 시각이 기록된다. 어제 저녁 22시에 먹었는데 기록은 오늘 오전 9시로 찍힌다. 정확성이 필요한 항우울제 복용 기록에서 이는 잠재적 오류다.
- **"병원 다음 날 약 빼먹는" 이 유저의 핵심 패인**: 앱을 여는 것만으로 "어제 놓쳤다"는 인앱 알림이나 배너가 없다. 직접 달력을 눌러봐야만 알 수 있다. 이 페르소나의 핵심 문제를 해결하지 못하고 있다.

### 잘 된 부분
- 소급 체크가 기술적으로 가능하다는 것 자체는 맞는 설계다.
- "미복용" 빨간 뱃지가 과거 날짜 상세에서도 보여서 어떤 약을 놓쳤는지 즉시 파악된다.
